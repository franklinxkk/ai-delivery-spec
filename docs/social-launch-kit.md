PRDs are vague, prototypes are not testable, and engineering handoff is
ambiguous. AI Delivery Spec adds routing, delivery tiers, readable PRD structure,
prototype testability rules, DDD/API handoff, QA gates, and domain modules.

## LinkedIn Post

AI coding agents are improving fast, but most product teams still break down
before code generation starts:

- PRDs are readable but not build-ready.
- Prototypes look complete but are not testable.
- Engineers receive feature names instead of business contracts.
- QA has no traceability from story to state to acceptance criteria.
- AI features ship without runtime, evaluation, or rollback contracts.

I open-sourced AI Delivery Spec to address this gap.

It is a product-side Spec-Driven Development framework for AI-era product teams:

- L0-L3 delivery tiers
- readable PRD structure
- executable prototype rules
- role path and E2E lifecycle matrices
- DDD/API/data handoff
- QA acceptance gates
- domain modules for CRM, traffic safety SaaS, and higher-education IT

Repo: https://github.com/franklinxkk/ai-delivery-spec
