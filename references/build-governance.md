# Build / Patch Governance

Use this file when prototypes or PRDs are generated by scripts, patch chains, transformations, or repeated HTML/JS edits.

## Patch vs Rebuild Decision

| Choose Patch Chain When | Choose Clean Rebuild When |
|---|---|
| mature prototype, small feature injection | repeated patches caused broken handlers |
| stable layout and data model | global architecture changed |
| change is localized | action/state names drifted across files |
| regression ledger exists | old baseline is unreliable |
| deadline favors minimal change | file is too large/fragile to patch safely |

## Script Naming

```text
_extract_*.py     reads and reports, no mutation
_transform_vX_Y.py  creates next prototype version
_fix_*.py        targeted repair
_verify_*.py     validation only
_build_*.py      deterministic artifact build
```

## Artifact Manifest

Every build/transform script should output or update:

```yaml
artifact_manifest:
  source_artifact: string
  output_artifact: string
  script: string
  version: string
  changed_scope: []
  preserved_paths: []
  added_paths: []
  de_scoped_paths: []
  verification:
    syntax: pass | fail
    action_handlers: pass | fail
    browser_paths: []
```

## Drift Detection

Run drift detection after three or more patch iterations or before handoff.

Check:
- duplicate/overridden functions;
- orphan `data-action`;
- removed views/actions/modals without de-scope;
- mock data no longer covers states;
- PRD action names no longer match prototype;
- generated files differ from source-of-truth file.

## Bundled Script

Use `scripts/extract_interaction_ledger.py` to create a first-pass ledger:

```powershell
python scripts/extract_interaction_ledger.py --input PROTOTYPE.html --output interaction-ledger.json
```

The script is a helper, not the final truth. Browser verification still matters for dynamic behavior.
