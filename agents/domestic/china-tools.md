# China Tools Adapter

| Tool | Recommended handoff |
|---|---|
| WorkBuddy | Put SKILL.md plus only the active Product Truth fragment/PRD section in workspace rules; save and validate one checkpoint per task. |
| Trae | Pin source order and forbidden invention in project rules; author one fragment or implement one vertical slice per task. |
| Qoder | Provide Coding Agent projection, exact repository baseline, AC IDs, and evidence paths. |
| Cursor | Put compact invariants in rules; attach only the current ID slice and linked contracts. |

For GLM 5.2, Qwen, DeepSeek, and Kimi, load the matching adapter. All tools must
report `CFL-*` when repository reality conflicts with Product Truth; none may
silently invent missing roles, pages, fields, states, permissions, or rules.

For large projects, never delegate a complete `product-truth.yaml` to a
subagent or one model response. Initialize progressive truth, write
`fragments/00-core.yaml` and one `fragments/MOD-*.yaml` at a time, then run the
deterministic compiler. Resume from `truth/index.yaml` after interruption.
